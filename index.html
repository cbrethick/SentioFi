What is SentioFi?

SentioFi is an emotion-intelligence layer embedded into financial platforms. It detects when retail investors are making decisions driven by panic, greed, or anxiety — and intervenes in real-time with evidence-based nudges that restore rational thinking. Not a robo-advisor. Not a trading bot. The missing human layer in every fintech stack.

System Architecture

SentioFi is a 4-layer intelligence stack — from raw user signals to compliant, explainable interventions. Every layer is independently deployable and privacy-preserving.

👤 RETAIL INVESTOR · Zerodha / Groww / Angel One / SentioFi+ App
↓ behavioral signals
Layer 1
Signal Ingestion
  • Chat NLP (FinBERT fine-tuned)
  • Behavioral telemetry: tap speed, scroll, session time
  • Search query semantics
  • Historical emotion footprint
Next.js SDK · FastAPI · WebSocket
→
Layer 2
Emotion Engine
  • Multi-class classifier (panic / greed / neutral / anxious)
  • Composite score: NLP(40%) + Behavior(40%) + Context(20%)
  • Temporal modeling — emotion velocity over session
  • Calibrated to SEBI suitability profiles
PyTorch · HuggingFace · ONNX Runtime
→
Layer 3
Intervention Logic
  • Threshold-based trigger rules (per-institution config)
  • Adaptive nudge library (52 evidence-based templates)

  • A/B tested for effectiveness & non-manipulation
LangChain · Redis · tRPC
→
Layer 4

  • On-device ONNX processing (privacy-first)

04 · Product Design

The Emotion Intelligence Layer

SentioFi operates invisibly until needed. In Calm Mode, it's frictionless. In Panic Mode or FOMO Mode, it becomes the pause button between impulse and loss.

📍 SCENARIO: Arjun, 29 · Retail Investor · Nifty drops 6% after RBI announcement at 9:15 AM
STEP 1 · Signal Detection

Arjun opens Zerodha. SentioFi detects: rapid screen jumps, loss-framing search ('crash', 'sell everything'), login at unusual hour.

STEP 2 · Emotion Classification

NLP model scores: HIGH ANXIETY (0.87). Behavioral velocity = 3× normal. Panic score threshold exceeded.

STEP 3 · Adaptive Intervention

"Pause before you act" card appears. Shows: Arjun's portfolio vs Nifty in 3 past crashes. Projected recovery at 12 months.

STEP 4 · Autonomy Preserved

Arjun can override after 30-sec pause. Choice logged with emotional context. User remains in full control.

✅ Result: Arjun pauses. Doesn't sell. Portfolio recovers +14% in 8 weeks. SentioFi saves avg ₹38,000 per prevented panic-sell event.
🎨 Design Principle: SentioFi adds friction where emotions dominate, and reduces friction where reason leads.

07 · Ethics & Privacy

Regulator-Ready by Design

Emotion data is among the most sensitive data that exists. SentioFi's architecture treats privacy as a first-class constraint, not an afterthought.

⚠ RISK: Manipulation — emotion data used to upsell
Strict prohibition: emotion signals may only trigger de-escalation nudges, never promotional content. Enforced at model level.
⚠ RISK: Privacy breach — emotional profiles leaked
On-device ONNX processing + differential privacy + zero-sale policy enforced via smart contracts. Raw data never leaves device.
⚠ RISK: Bias — model trained on non-representative data
Diverse training data + quarterly bias audits + fairness constraints embedded in classifier loss function.
⚠ RISK: Consent fatigue — users blindly accept terms
Layered visual consent UX; emotion features off-by-default; can be disabled per-session with 2 taps.
⚠ RISK: Over-reliance — users defer all decisions to AI
Mandatory human confirmation for all high-value trades regardless of emotional state. AI advises; human decides.
⚠ RISK: Regulatory non-compliance
Aligned with SEBI suitability framework, RBI Digital Lending guidelines, GDPR/DPDPA, SEC/FINRA Reg BI, and ISO 42001 AI standard.
🔒 Privacy-by-Design Principles
On-device NLP processing Differential privacy Explicit opt-in controls No third-party data sale Right-to-forget in 2 clicks Immutable audit log
08 · Getting Started

Setup & Installation

Prerequisites: Node.js 20+, Python 3.12+, Docker, Git. All dependencies are free and open-source.

## 1. Clone the repository
$ git clone https://github.com/causecraft-ai/sentiofi.git
$ cd sentiofi
## 2. Install Next.js frontend (App Router + TypeScript)
$ cd apps/web
$ npm install
$ cp .env.example .env.local
$ npm run dev # → http://localhost:3000
## 3. Set up the Python AI / emotion engine
$ cd services/emotion-engine
$ python -m venv .venv && source .venv/bin/activate
$ pip install -r requirements.txt
$ python scripts/download_models.py # FinBERT + LSTM weights
$ uvicorn main:app --reload --port 8000
## 4. Run everything with Docker Compose
$ docker compose up --build
 
## Services:
# → Frontend (Next.js) http://localhost:3000
# → API Gateway (FastAPI) http://localhost:8000
# → Supabase (local) http://localhost:54323
# → Kafka UI http://localhost:8080
# → Grafana Dashboard http://localhost:3001
## Required environment variables (all free tier)
 NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
 NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
 GROQ_API_KEY=your_groq_key # free at console.groq.com
 UPSTASH_REDIS_URL=your_redis_url # free at upstash.com
 HF_TOKEN=your_hf_token # free at huggingface.co
